### 方式 2：手动部署（完全掌控）⭐⭐⭐⭐

#### 步骤 1：推送代码到 GitHub（2 分钟）

```bash
# 初始化 Git（如果还没有）
git init

# 添加所有文件
git add .

# 提交
git commit -m "Ready for deployment"

# 创建 GitHub 仓库（访问 https://github.com/new）
# 然后添加远程仓库
git remote add origin https://github.com/你的用户名/blueprint-saas.git

# 推送
git branch -M main
git push -u origin main
```

#### 步骤 2：在 Railway 部署（10 分钟）

1. **访问 Railway**
   - 🔗 https://railway.app/
   - 用 GitHub 登录

2. **创建项目**
   - 点击 "New Project"
   - 选择 "Deploy from GitHub repo"
   - 选择你的仓库

3. **添加数据库**
   - 点击 "New" → "Database" → "PostgreSQL"

4. **配置后端**（server 目录）
   ```bash
   DATABASE_URL=${{Postgres.DATABASE_URL}}
   JWT_SECRET=<生成随机字符串>
   NODE_ENV=production
   PORT=5000
   ```

5. **配置前端**（根目录）
   ```bash
   VITE_API_URL=https://后端地址.railway.app/api
   ```

6. **生成域名并访问**

详细步骤请查看 **DEPLOY_NOW.md**

## 💡 重要提示

### 1. JWT Secret 生成
```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

### 2. 环境变量配置

**后端必需**：
- DATABASE_URL
- JWT_SECRET
- NODE_ENV=production
- PORT=5000

**前端必需**：
- VITE_API_URL=https://后端地址/api

### 3. CORS 配置
后端需要配置前端域名：
```bash
CORS_ORIGIN=https://前端地址.railway.app
```
