## 🎯 解决方案

### 问题
Docker 前端容器的端口映射在 Windows WSL2 环境下失败

### 解决方案
使用 Vite 本地开发服务器代替 Docker 容器

### 优点
- ✅ 热重载支持 - 修改代码立即生效
- ✅ 更快的启动速度
- ✅ 更好的开发体验
- ✅ 更容易调试
- ✅ 不依赖 Docker 网络

前端开发服务器正在后台运行（进程 ID: 3）

### 停止前端服务器
如果需要停止前端服务器，可以：
1. 在终端中按 `Ctrl+C`
2. 或者关闭运行 `npm run dev` 的终端窗口

### 重启前端服务器
```bash

```

### 查看后端日志
```bash
docker logs -f blueprint_backend
```

### 停止所有 Docker 服务
```bash
docker-compose down
```

## 💡 开发提示

### 前端开发
- 修改 `components/` 中的文件会自动重新加载
- 修改 `services/` 中的文件会自动重新加载
- 浏览器控制台可以查看错误和日志

### 后端开发
- 修改后端代码后需要重启容器：
  ```bash
  docker-compose restart backend
  ```

### 数据库操作
- 连接数据库：
  ```bash
  docker exec -it blueprint_postgres psql -U postgres -d blueprint_saas
  ```
