# 🎉 SaaS 版本使用指南

## ✅ 已集成的功能

你的应用现在已经是完整的 SaaS 版本了！包含：

1. **用户认证系统** ✅
   - 用户注册
   - 用户登录
   - JWT Token 认证
   - 自动登出

2. **数据持久化** ✅
   - PostgreSQL 数据库
   - 用户数据隔离
   - 跨设备同步

3. **完整的 API** ✅
   - 用户注册/登录接口
   - 项目管理接口
   - RESTful API 设计

## 🚀 如何使用

### 第一步：访问应用

打开浏览器，访问：**http://localhost:3000**

### 第二步：注册账户

首次访问会看到登录/注册弹窗：

1. 点击"还没有账户？立即注册"
2. 填写信息：
   - 姓名：你的名字
   - 邮箱：your@email.com
   - 密码：至少6位
3. 点击"注册"按钮

### 第三步：开始使用

注册成功后会自动登录，你可以：

- ✅ 创建项目
- ✅ 管理模块和实体
- ✅ 使用 AI 功能
- ✅ 数据自动保存到数据库

### 第四步：登出和重新登录

- 点击右上角的"登出"按钮
- 下次访问时输入邮箱和密码登录
- 你的数据会自动加载

## 🔐 用户系统特性

### 数据隔离
- 每个用户只能看到自己的项目
- 数据完全隔离，安全可靠

### 跨设备同步
- 在任何设备登录同一账户
- 数据自动同步

### 安全认证
- 密码加密存储（bcrypt）
- JWT Token 认证
- 7天自动登录

## 📊 数据存储

### 本地版 vs SaaS 版

| 特性 | 本地版 | SaaS 版 |
|------|--------|---------|
| 数据存储 | localStorage | PostgreSQL |
| 多用户 | ❌ | ✅ |
| 跨设备 | ❌ | ✅ |
| 数据备份 | 手动 | 自动 |
| 团队协作 | ❌ | 可扩展 |

## 🎯 API 端点

### 认证相关
```
POST /api/auth/register  - 注册新用户
POST /api/auth/login     - 用户登录
```

### 项目管理
```
GET    /api/projects     - 获取项目列表
POST   /api/projects     - 创建新项目
PUT    /api/projects/:id - 更新项目
DELETE /api/projects/:id - 删除项目
```

## 🔧 开发者信息

### 查看数据库

```powershell
# 进入数据库容器
docker exec -it blueprint_postgres psql -U postgres blueprint_saas

# 查看用户表
SELECT * FROM users;

# 查看项目表
SELECT * FROM projects;

# 退出
\q
```

### 查看日志

```powershell
# 后端日志
docker-compose logs -f backend

# 前端日志
docker-compose logs -f frontend

# 数据库日志
docker-compose logs -f postgres
```

### 重启服务

```powershell
# 重启所有服务
docker-compose restart

# 只重启前端
docker-compose restart frontend

# 只重启后端
docker-compose restart backend
```

## 🌐 部署到生产环境

当你准备部署到服务器时：

1. **购买服务器**（阿里云/腾讯云）
2. **配置域名**
3. **上传代码**
4. **修改 .env 配置**：
   ```env
   FRONTEND_URL=https://your-domain.com
   API_URL=https://api.your-domain.com/api
   ```
5. **启动服务**：
   ```bash
   docker-compose up -d
   ```
6. **配置 HTTPS**（Let's Encrypt）

详见：`部署指南-生产环境.md`

## 🆘 常见问题

### Q: 忘记密码怎么办？
A: 目前需要手动重置数据库。后续可以添加"忘记密码"功能。

### Q: 可以修改邮箱吗？
A: 目前不支持，后续可以添加个人资料编辑功能。

### Q: 数据会丢失吗？
A: 数据存储在 PostgreSQL 数据库中，定期备份即可。

### Q: 如何备份数据？
A: 运行 `.\scripts\backup.ps1` 自动备份数据库。
